How these are calculated
------------------------

Length and response volume (`len_*`)
- Simple transcript-derived counts and rates.
- Examples:
  - `len_token_count`: number of tokens in the response
  - `len_mean_utterance_length`: average tokens per utterance
  - `len_syllables_per_second`: syllables divided by speech duration

Lexical diversity and richness (`lex_*`)
- Computed from tokenized / lemmatized transcript text.
- Examples:
  - `lex_type_token_ratio`: unique surface forms divided by total tokens
  - `lex_lemma_type_token_ratio`: unique lemmas divided by total tokens
  - `lex_mattr_10` / `lex_mattr_20`: moving-average type-token ratio over 10 / 20 token windows
  - `lex_brunet` / `lex_honore`: standard lexical-richness statistics

Discourse repetition and coherence (`disc_*`)
- Based on repetition patterns and utterance-to-utterance semantic similarity.
- Examples:
  - `disc_repeated_bigram_ratio`: repeated bigrams divided by total bigrams
  - `disc_immediate_repetition_count`: immediate repetition count
  - `disc_mean_pairwise_utt_similarity`: mean semantic similarity between utterances
  - `disc_semantic_drift`: how far utterances move semantically across the response

Core syntax (`syn_*`)
- Derived from POS/dependency-style parsing or CLAN/Stanza-backed syntax extraction.
- Examples:
  - `syn_pronoun_ratio`: pronoun tokens divided by all tokens
  - `syn_clause_density`: clause-like relations relative to utterance length
  - `syn_mean_dependency_length`: average dependency span length
  - `syn_max_tree_depth`: deepest dependency tree depth observed

Speech graphs (`graph_*`)
- Tokens are represented in a transition graph; graph statistics summarize structure.
- Examples:
  - `graph_density`: observed edges relative to possible edges
  - `graph_average_degree`: mean node degree
  - `graph_self_loop_ratio`: proportion of self-loops / repeated transitions

Pause timing (`pause_*`)
- Derived from audio silence structure and filled-pause behavior.
- Examples:
  - `pause_mean_duration`: average pause duration
  - `pause_pause_to_word_ratio`: pause burden relative to word count
  - `pause_speaking_to_total_ratio`: speaking time divided by total recording time

Acoustic and paralinguistic blocks (`ac_*`, `par_*`)
- Computed from waveform-level audio features.
- `ac_*` is the main universal acoustic block used in Phase 1.
- `par_*` is the richer Phase 2 paralinguistic extension.
- Examples:
  - eGeMAPS loudness, jitter, spectral flux, MFCC functionals
  - RMS energy summaries
  - zero-crossing and pitch dispersion statistics

Richer lexicosyntax and production structure (`sx_*`, `pr_*`)
- Added to match richer handcrafted-feature papers more closely.
- `sx_*` expands the higher-level lexical/syntactic summary layer.
- `pr_*` approximates Balagopalan-style production / phrase structure cues.

Task-specific semantic blocks (`pd_*`, `rd_*`, `fc_*`, `ft_*`, `sr_*`)
- These are Phase 2 features designed for cleaner, more interpretable task slices.
- `pd_*`: picture-description content-unit style features
- `rd_*`: reading-reference and deviation features
- `fc_*`: conversation / topic features
- `ft_*`: verbal fluency productivity, switching, intrusions
- `sr_*`: story-reference / narrative alignment features

Interpretation guidance
-----------------------

- Core features are the broad multilingual baseline:
  - `len`, `lex`, `disc`, `syn`, `graph`, `pause`, `ac`
- Phase 2 also adds extra non-task-specific core features:
  - deeper acoustic / paralinguistic detail
  - richer lexicosyntax / production structure
- These live under:
  - `par`, `sx`, `pr`
- Phase 2 task-specific features are:
  - `pd`, `rd`, `fc`, `ft`, `sr`
- So Phase 2 is not only task-specific expansion; it also materially enlarges the core acoustic / lexical / syntactic inventory.

- If the goal is broad multilingual interpretability:
  - focus first on the full core inventory
- If the goal is task-clean clinical interpretability:
  - the Phase 2 task-specific blocks are the more important additions
