Paper vs current benchmark/source data

Comparison basis
- "Paper" is transcribed from Table 1 in `2024_multiconad_multilingual_alzheimer_shakeri_farmanbar_balog_annotated.pdf`.
- Table 1 is mostly source-level counts before the 80/20 train/test split.
- The one exception is NCMMSE/NCMMSC: the paper row is train-only because the paper did not use the labeled test set.
- "Current repo source pool" below is computed from the live extracted JSONL files in `data/processed/extracted/`.
- "Current cleaned benchmark" below is computed from the live cleaned train/test JSONL files in `data/processed/cleaned/`.
- This file is independent of the Phase 1 / Phase 2 feature experiments.

Diagnosis bucket mapping used for current repo counts
- `DM`: dementia-like labels that are not explicitly `AD` in the raw source files, e.g. `Dementia`, `D`, `Memory`, `Vascular`, `Pick's`, `Aphasia`
- `AD`: explicit AD-like labels, e.g. `AD`, `ProbableAD`, `PossibleAD`, `Alzheimer's`, `DTA`
- `HC`: healthy/control-like labels, e.g. `HC`, `Control`, `CTRL`, `NC`, `H`
- `Unknown`: rows still unlabeled in the raw extracted pool

Key observations
- NCMMSE is no longer stale here. The current source pool includes the labeled test subset, so the repo has `399` labeled NCMMSE rows versus the paper's `280` train-only row.
- English Taukdial and Chinese Taukdial are separated in the current repo, whereas the paper reports one English row and does not break out the Chinese test portion this way.
- Delaware, DS3, and Taukdial are materially larger in the current repo because the repo keeps file-level/session-level datapoints, not a paper-compatible participant-level subset.
- WLS remains weakly labeled in the repo. The current source pool is `262 DM / 1106 HC`; the paper row reports `263 AD / 1106 HC`.
- The cleaned benchmark totals are expected to differ from the raw source-pool totals because unlabeled rows are dropped and the cleaning scripts apply text-length filtering before the train/test split.
